# i18n Design — Parody Critics (Phase 1)
Date: 2026-03-03

## Objetivo

Añadir soporte EN/ES a la UI de Parody Critics con una arquitectura extensible a N idiomas futuros sin cambios de código.

---

## Arquitectura: JSON files + loader

```
static/locales/
  es.json     ← strings en español (idioma base)
  en.json     ← strings en inglés
static/js/
  i18n.js     ← loader + t('key') helper
```

### Por qué esta arquitectura
- Añadir un tercer idioma = crear un archivo JSON, zero cambios en código
- Un traductor puede trabajar sin saber programar
- Sin dependencias externas, sin build step
- Persiste en localStorage

### i18n.js — contrato público
```javascript
window.i18n = {
  current: 'es',            // idioma activo
  t(key),                   // traduce un key
  setLanguage(lang),        // cambia idioma + recarga strings
  init(),                   // detecta localStorage → carga locale
}
```

### data-i18n en HTML (strings estáticos)
```html
<button data-i18n="generate.button">Generar Crítica</button>
```
`setLanguage()` recorre todos los elementos con `data-i18n` y actualiza el texto.

### t('key') en JS (strings dinámicos)
```javascript
showToast(t('toast.critic_generated'))
showError(t('error.model_unavailable'))
```

## Selector de idioma

Ubicación: header de la UI, junto al logo.

```html
<select id="lang-selector">
  <option value="es">ES</option>
  <option value="en">EN</option>
</select>
```

Valor por defecto: localStorage → `es` si no hay preferencia guardada.

## Alcance Phase 1 — strings a traducir

**`index.html`** (~60-80 keys con `data-i18n`):
- Header, tabs, labels de formularios
- Botones: Generate, Add to Cart, Clear Cart
- Mensajes de estado vacío, tooltips, placeholders

**`app.js`** (~80-100 keys con `t('key')`):
- Toasts de éxito/error
- Mensajes de loading
- Confirmaciones de borrado
- Placeholders generados dinámicamente

**Soul Wizard** — mismo sistema, incluido en Phase 1.

**Lo que NO se traduce:**
- Nombres y descripciones de personajes (contenido del usuario)
- Críticas generadas (idioma del LLM)
- Títulos de Jellyfin, géneros, metadatos externos

Estimado total: ~150-180 keys.

## Soul Wizard — idioma del contenido generado

Toggle EN/ES dentro del wizard, independiente de la UI:
- Por defecto: idioma activo de la UI
- Se pasa como parámetro al prompt de `soul_generator.py`
- **No se persiste en DB** — el soul ya contiene el texto en el idioma correcto; leerlo es suficiente

```python
# soul_generator.py — una línea añadida al system prompt
lang_instruction = "Respond in English." if language == "en" else "Responde en español."
```

## LLM — System prompt en inglés

### Hipótesis
Los LLMs tienen 60-90% de training data en inglés. Prompts en inglés activan más capacidad del modelo → mejor instruction-following, mejor calibración ideológica.

### Benchmark previo a implementación

3 variantes, mismo personaje + misma película:

| Variante | System prompt | Soul | Output |
|----------|--------------|------|--------|
| A | Español | Español | Español |
| B | Inglés | Español | Español (`Respond in Spanish`) |
| C | Inglés | Inglés | Inglés → traducir a Español |

Métricas: coherencia ideológica, creatividad, calibración del rating, naturalidad del texto.

### Decisión post-benchmark
- **Diferencia marginal** → mantener español, simplificar
- **Inglés notablemente superior** → migrar SYSTEM_BLOCK a inglés + añadir traducción opcional

### Si se implementa traducción

Opciones evaluadas:
- **DeepL API** — mejor calidad para texto literario/creativo, 500k chars/mes gratis, ~300ms
- **LLM auto-traducción** — preserva tono y humor, añade 2-5s de latencia

Configurable vía `.env`: `TRANSLATE_OUTPUT=true` / `TRANSLATE_PROVIDER=deepl`

### Cambio inmediato en prompt_builder.py (independiente del benchmark)

El SYSTEM_BLOCK recibe el idioma de salida como parámetro:
```python
SYSTEM_BLOCK_EN = """You are a parody film criticism system.
Your rating ALWAYS reflects the character's ideological perspective...
Respond in {output_language}."""
```

## Lo que NO cambia

- La lógica de generación de críticas
